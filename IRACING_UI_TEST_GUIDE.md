# iRacing UI 테스트 가이드 (Mock 데이터)

## 🎯 테스트 목표

Mock 데이터를 생성하여 메타 리포트와 BoP 알림 UI를 테스트합니다.

## 📋 테스트 절차

### 1단계: Mock 데이터 생성

1. **iRacing 페이지 접속**
   - `/iracing` 페이지로 이동
   - "메타 리포트" 탭 선택

2. **Mock 데이터 생성**
   - 상단 우측의 "Mock 10개", "Mock 20개", "Mock 50개" 버튼 중 선택
   - 권장: 처음엔 "Mock 10개"로 시작
   - 생성 완료 알림 확인

3. **자동 리포트 조회**
   - Mock 데이터 생성 후 자동으로 리포트 조회됨
   - 시리즈 ID가 없으면 첫 번째 시리즈로 자동 설정

### 2단계: 메타 리포트 테스트

1. **시리즈별 리포트 조회**
   - 시리즈 ID 입력: `123` (GT3 Challenge) 또는 `456` (Formula 3 Championship)
   - 기간 선택: 7일, 14일, 30일, 60일
   - "조회" 버튼 클릭

2. **트랙별 리포트 조회**
   - 시리즈 ID: `123`
   - 트랙 ID: `1` (Watkins Glen), `2` (Silverstone), `3` (Nürburgring)
   - "조회" 버튼 클릭
   - **확인 사항**: iRating 구간별 랩타임에 트랙 이름이 표시되는지 확인

3. **차량별 통계 카드 확인**
   - 승률, 픽률, Top 5율 확인
   - iRating 구간별 랩타임 (트랙 지정 시)
   - 차량별 정렬 (승률 기준)

### 3단계: BoP 패치 알림 테스트

1. **BoP 패치 알림 탭 선택**
   - "BoP 패치 알림" 탭 클릭

2. **알림 조회**
   - 시리즈 ID (선택): `123` 또는 비워두기
   - 임계값: `20` (기본값)
   - "알림 조회" 버튼 클릭

3. **알림 카드 확인**
   - 급상승/급하락 알림 표시
   - 승률, 픽률, Top 5율 변화 확인
   - 색상 구분 확인 (녹색: 상승, 빨간색: 하락)

## 🔧 문제 해결

### "데이터가 없습니다" 메시지
- Mock 데이터를 먼저 생성했는지 확인
- 시리즈 ID가 올바른지 확인 (123 또는 456)
- 기간을 60일로 늘려보기

### "iRating 구간별 랩타임"이 표시되지 않음
- 트랙 ID를 입력했는지 확인
- 트랙 ID가 올바른지 확인 (시리즈 123: 1, 2, 3 / 시리즈 456: 10, 11)

### BoP 알림이 없음
- Mock 데이터가 충분히 생성되었는지 확인 (최소 20개 이상 권장)
- 임계값을 낮춰보기 (10% 또는 15%)

## 🎨 UI 개선 확인

- 그라디언트 스타일 적용 확인
- 카드 레이아웃 및 호버 효과
- 반응형 디자인 (모바일/태블릿/데스크톱)
- 로딩 및 에러 상태 처리
